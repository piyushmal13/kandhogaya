# Skill: SEO & GEO Domination Engine

## Goal
Optimize codebase architecture, HTML structure, routing, and metadata layout to rank at the top of organic search engines (SEO) and generative AI search models (GEO).

## Instructions
1. **Semantic HTML Enforcement**: Every page view must utilize structural layout elements (`<header>`, `<main>`, `<section>`, `<article>`, `<footer>`). Headings must follow a strict sequential hierarchy (`<h1>` down to `<h6>`) with zero omissions.
2. **Generative Engine Optimization (GEO)**: 
   - Ensure a high density of exact, authoritative technical industry terms (e.g., naming specific server hubs like "Equinix NY4", infrastructure types, and protocol versions).
   - Format dense information, stats, and configurations using clean typographic Markdown/HTML tables. Generative LLM scrapers prioritize organized data arrays over loose marketing prose.
3. **Structured Data Injections**: Inject complete JSON-LD packages targeting `Organization`, `WebSite` (with `SearchAction`), and `SiteNavigationElement` configurations into core route templates.
4. **Crawlable Navigation Paths**: Never hide primary domain routes behind pure JavaScript state engines. Ensure the application navbars and footers render native link elements to build an optimized crawl map.

## Constraints
- Never include broken metadata strings or empty header tags.
- Never use generic filler text ("Lorem Ipsum") in newly generated solution pages; always draft precise, contextual industry text.
